# Deformable Infantry 主动悬挂控制

## 项目背景与需求

### 核心目标

为 deformable infantry v2（舵轮）和 omni（全向轮）实现**主动悬挂控制**。关节通过连杆控制四个轮组和底盘之间的角度，可以改变轮距。v2 和 omni 的关节控制相同。

**核心效果**: 轮组在上坡或过坎时悬空了，关节可以使轮组触地。

### 主动悬挂优势

被动弹簧只能存储/释放能量，而电机可以**主动注入能量**并实时改变特性。

### 系统自由度分析

四个悬挂电机控制的自由度：

| 自由度 | 含义 |
|--------|------|
| z | 底盘高度 |
| theta (pitch) | 俯仰 |
| phi (roll) | 横滚 |
| **冗余 1 DOF** | 4 个执行器 - 3 个姿态 DOF = 1 个自由度用于优化 |

### 分层控制架构（完整目标）

```
┌─────────────────────────────────────────┐
│  Layer 3: 轮压优化 (利用冗余DOF)         │  ← 最大化底盘性能
├─────────────────────────────────────────┤
│  Layer 2: 姿态控制 (z, theta, phi)      │  ← IMU 反馈保持底盘水平
├─────────────────────────────────────────┤
│  Layer 1: 阻抗控制 (每条腿独立)          │  ← 柔顺接地
└─────────────────────────────────────────┘
```

### Layer 1: 虚拟阻抗控制（基础层）

让每个电机模拟一个可调弹簧-阻尼器：
```
tau_i = K_i * (l0 - l_i) + D_i * (-dl_i/dt) + tau_gravity
```
- K_i — 虚拟刚度（实时可调）
- D_i — 虚拟阻尼
- tau_gravity — 重力补偿项（mg/4）

### Layer 2: 姿态控制

用 IMU 反馈，通过 PD 控制维持底盘水平。将姿态误差映射到每条腿的长度修正：
```
delta_l_i = delta_z + x_i * delta_theta + y_i * delta_phi
```

### Layer 3: 轮压优化

利用冗余自由度，根据运动指令动态分配轮压。本质上是一个可实时求解的小规模 QP 问题：
```
minimize   sum(N_i - mg/4)^2
subject to N_i >= |F_traction_i| / mu,  sum(N_i) = mg
```

### 动态补偿能力

| 场景 | 被动悬挂 | 主动悬挂 |
|------|----------|----------|
| 急加速 | 点头 | 前馈提升前轮力，抑制点头 |
| 急转弯 | 内侧轮压减小 | 前馈补偿，维持均压 |
| 过障碍 | 被动吸收，有延迟 | 提前调整 |

## 实现方案：ADRC 方案

### 技术决策

在 `DeformableJointController` 内部用 ADRC 替换 PID 级联，实现双模式（正常跟踪 + 悬挂柔顺），不依赖独立的 `ActiveSuspension` 组件。扫频辨识适配 ADRC。

## ADRC 如何实现弹簧悬挂

### ADRC 各组件与弹簧-阻尼器的对应关系

| ADRC 组件 | 参数 | 悬挂物理含义 |
|-----------|------|-------------|
| **NLESF k1** | 位置误差增益 | **虚拟弹簧刚度** — k1 越大 = 弹簧越硬 |
| **NLESF k2** | 速度误差增益 | **虚拟阻尼系数** — k2 越大 = 阻尼越强 |
| **NLESF alpha1** | 位置非线性指数 | alpha1 < 1: 小误差响应强（精密），大误差饱和（安全） |
| **NLESF alpha2** | 速度非线性指数 | alpha2 < 1: 类似，有利于冲击吸收 |
| **ESO w0** | 观测器带宽 | **扰动跟踪速度** — w0 低 = 无法跟踪快速扰动 = 自然柔顺 |
| **ESO z3** | 总扰动估计 | 自动包含重力 + 地面反力 + 摩擦 — 天然重力补偿 |
| **TD td_r** | 轨迹跟踪速度 | 悬挂模式降低 = 更平滑的目标过渡 |
| **输出限幅** | output_min/max | **最大力矩** — 低限幅 = 更柔顺 |

### 双模式 ADRC 参数设计

**正常模式（刚性跟踪）**:
```
TD:    td_r = 50       → 快速跟踪目标
ESO:   w0 = 250        → 高带宽，快速估计扰动并补偿
NLESF: k1 = 30, k2 = 17 → 高增益 → 硬弹簧 + 高阻尼
限幅:  ±200 N·m        → 全力矩
→ 效果: 精确位置跟踪，像锁死的悬挂
```

**悬挂模式（柔顺弹簧）**:
```
TD:    td_r = 10        → 慢速跟踪，平滑响应
ESO:   w0 = 50~100      → 低带宽，无法快速跟踪冲击
NLESF: k1 = 5~10, k2 = 3~8 → 低增益 → 软弹簧 + 适度阻尼
限幅:  ±50 N·m          → 限制力矩输出
→ 效果: 柔顺接地，像弹簧悬挂
```

### 为什么 ADRC 悬挂模式等效于阻抗控制

ADRC 控制律:
```
u = (k1 * fal(e1, alpha1, delta) + k2 * fal(e2, alpha2, delta) - z3) / b0
```

展开分析:
- `k1 * fal(e1)` → **弹簧力**: 位置偏离目标 → 产生恢复力
- `k2 * fal(e2)` → **阻尼力**: 速度偏离零 → 产生阻尼力
- `-z3 / b0` → **扰动补偿**: 自动补偿重力、摩擦等（替代显式重力补偿）

这与阻抗控制 `τ = K*(x0-x) + D*(-dx/dt) + τ_gravity` 本质相同:
- `K` ↔ `k1/b0`
- `D` ↔ `k2/b0`
- `τ_gravity` ↔ `-z3/b0`（ESO 自动估计）

### 悬空自动伸展机制

1. 目标 = min_angle (20°, 最大伸展)
2. 轮组离地 → 地面反力消失 → ESO z3 减小
3. 位置误差 e1 = target - current ≠ 0 → NLESF 产生伸展力
4. 悬挂模式 k1 较低，力矩有限 → 温和伸展（不会猛冲）
5. 轮组触地 → 地面反力出现 → ESO z3 增大 → 新平衡
6. 如果冲击过快 → ESO 来不及跟踪（低 w0）→ 关节自然回弹吸收

### ESO z3 作为接地检测信号

z3 直接反映关节上的"总负载"：
- **触地时**: z3 大（重力 + 地面反力）
- **悬空时**: z3 小（仅重力分量）
- **冲击时**: z3 突变

→ 可以用 z3 做地面接触检测，无需额外力矩传感器回路

## 架构

```
DeformableChassis (chassis_controller)
  ├── 输出: target_physical_angle（轨迹规划）
  ├── 输出: suspension_mode（true = 悬挂模式）
  └── 输出: suspension_torque（可选前馈，ADRC 可不用）

DeformableJointController (ADRC 内核, 4 实例)
  ├── 输入: measurement_angle, measurement_velocity
  ├── 输入: setpoint_angle, setpoint_velocity（ADRC 内部由 TD 生成速度）
  ├── 输入: suspension_mode → 切换 ADRC 参数组
  ├── 输入: suspension_torque → 可选前馈加到输出
  ├── 内部: TD → ESO → NLESF（双模式参数）
  ├── 输出: control_torque
  └── 输出: eso_z3（供扫频记录/调试）
```

**优势**: 保持与现有 `DeformableChassis` 完全相同的接口约定，只需替换内部控制律。

## 扫频辨识适配 ADRC

### 目标

用扫频数据辨识 ADRC 核心参数，特别是:
- **b0** (标称控制增益) — 最关键参数
- 系统带宽 → 确定合适的 w0
- 不同工作角度下的特性变化 → 确定是否需要增益调度

### 扫频流程（不变）

```
SweepController → 目标角度（chirp 信号）
↓
DeformableJointController (ADRC) → 控制力矩
↓
Hardware → 电机反馈
↓
SweepRecorder → CSV 数据
```

### 扫频记录需增加的信号

在 Recorder 中新增:
- `eso_z3`: ESO 扰动估计（用于分析负载特性）
- `eso_z2`: ESO 速度估计（与实际速度对比验证 ESO 收敛性）

### 从扫频数据辨识 b0

b0 的物理含义: `角加速度 = b0 * u + 扰动`

辨识方法:
1. 从 CSV 数据中取 `control_torque (u)` 和 `actual_angle`
2. 对 actual_angle 做二阶数值微分得到角加速度
3. 用最小二乘拟合: `b0 = mean(angular_acceleration / control_torque)` (扰动项会被平均掉)
4. 或用频域方法: Bode 图低频段增益 ≈ b0

### 从 b0 推导悬挂参数

已知 b0 和期望特性:
```
期望弹簧刚度 K_desired [N·m/rad] → k1 ≈ K_desired * b0
期望阻尼系数 D_desired [N·m·s/rad] → k2 ≈ D_desired * b0  
ESO 带宽 w0 → 设为系统带宽的 3~5 倍（从 Bode 图确定系统带宽）
```

## 分阶段规划

### Phase 1: ADRC 基础悬挂
- 改造 `DeformableJointController`: PID → ADRC 内核 + 双模式
- 暴露 ESO z3 输出
- 扫频记录器增加 z3/z2 信号
- 用扫频数据辨识 b0，确定悬挂参数

### Phase 2: 增强悬挂
- ESO z3 做接地检测 → 悬空时主动调参加速伸展
- 底盘 IMU 姿态反馈 → 调整各关节目标角度

### Phase 3: 轮压优化
- 冗余 DOF 力分配
- 运动指令驱动的法向力优化
